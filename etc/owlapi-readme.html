Core OWL Library Bundle

This is one of the core Protégé plugins and provides access to the owl api library for Protégé owl bundles.

Version info

This version of the OWLAPI includes patches that have not yet been included in the official OWL API svn.

v4.1.0 Build 209 - 28 Sep, 2010

  • Fixes a problem where Manchester OWL Syntax files with UTF-8 characters are corrupted on read. See the OWLAPI GForge ticket.

v4.1.0 Build 209 Update - 19 Sep, 2010

  • Added a fix for a typo in the owl api for the NMTOKEN datatype. See the OWL API GForge ticket.
  • Allow the value of a facet space to have datatypes other than string. The correctness of this change is still under investigation.

v4.1.0 Build 209 Update - 16 Sep, 2010

  • Fixes a problem where ontologies with UTF-8 characters are corrupted on the save operation. See the OWLAPI GForge ticket.

v4.1.0 Build 209 - 3 Sep, 2010

This was a fairly significant milestone since the last build. It was at this point that Protege 4.1 moved to the OWL API 3.1.

v4.1.0 Build 102 Update - 14 Apr, 2010

We are now at the OWL API svn revision 1451 which is beyond the svn revision of the official release.

  • Fix for a problem with the multiply and add built-ins that prevent Protege from parsing rules with these built-ins.

v4.1.0 Build 102 Update - 22 Mar, 2010

  • Certain inferences in the exported inferred ontology missed being included in the Protégé Build 102 version of the OWL API. This update includes that functionality.

v4.1.0 Build 102 (19 Mar, 2010)

  • See the Protégé 4 release notes.
